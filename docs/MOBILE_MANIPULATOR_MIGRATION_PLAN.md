# PhysHSI 架构迁移计划：移动抓取 (Mobile Grasping) MVP

## 1. 迁移目标与核心方法论

**当前目标**：为了降低初期迁移的风险，我们将复杂的“拿-搬-放”全流程缩小为**“移动抓取” (Mobile Grasping / Pick Task)** 的最小可行性验证 (MVP)。重点验证底盘移动与机械臂抓取的协同控制。

**保留的核心方法论**：
1. **AMP (Adversarial Motion Priors)**：使用对抗性先验，约束机器人生成平滑、协同的（非生硬的）控制策略。
2. **阶段性状态初始化 (Stage-wise Initialization / RSI)**：通过在任务的不同阶段进行初始化，解决长程 (Long-horizon) 任务梯度稀疏的问题。

---

## 2. 核心需求与准备工作清单

### 2.1 硬件/仿真资产 (Assets)
- [ ] **URDF 文件**：需要包含轮式底盘（速度控制/虚根节点）和单机械臂（如 UR5, Franka 等）以及两指/多指夹爪。
- [ ] **关节映射定义**：整理新机器人的 DOF 列表，区分底盘驱动、机械臂关节与夹爪致动器。

### 2.2 专家数据 (Expert Data) —— MVP 级需求
由于目标缩减为“抓取”，专家数据的准备工作大幅简化。
- **需求**：至少 50 条包含 `[底盘位姿, 机械臂关节角, 夹爪开合度, 目标物体状态]` 的短程时间序列数据。
- **轨迹内容**：仅需记录 **“机器人在物体远处 -> 底盘靠近 -> 伸出手臂 -> 夹爪闭合并使物体发生位移（拿稳）”** 这一过程。
- **获取途径建议**：
  1. **脚本/规划器生成**：底盘执行 A* 导航，到达固定身位后执行 MoveIt! 预设的抓取动作。
  2. **遥操作采集 (Teleop)**：使用手柄人工控制。

## 3. 代码重构细节 (Refactoring Details)

### Phase 1: 基础环境剥离
在 `legged_gym/envs/` 下创建新的 MVP 目录（如 `envs/mobile_pick/`）。

1. **底层类解耦**：
   - 移除 `legged_robot.py` 中关于“脚部接触”、“空中时长”的硬编码。
2. **动作空间重构 (`_compute_torques`)**：
   - **机械臂与夹爪**：PD 控制（高刚度 `P`，适中 `D`）。
   - **轮式底盘**：将网络输出映射为底盘的目标线速度 (vx, vy) 和角速度 (wz)。

### Phase 2: 观测空间与 MVP 奖励函数 (Obs & Rewards)

1. **观测空间 (Observation Space)**
   - **底盘状态**：线速度、角速度、相对于目标物体的 Heading Error。
   - **机械臂状态**：关节位置、速度。
   - **任务感知**：夹爪（End-Effector）相对于目标物体的三维偏差向量、夹爪当前的开合状态。

2. **任务奖励函数 (Task Rewards - Pick Focused)**
   - **`approach_reward` (靠近)**：底盘稳定靠近物体。
   - **`reaching_reward` (伸臂)**：夹爪末端距离目标物体的指数衰减奖励。
   - **`grasp_reward` (抓取)**：夹爪处于闭合状态，且物体 Z 轴高度大于初始高度（成功离地）。
   - **惩罚项**：底盘碰撞惩罚、机械臂奇异性惩罚。

### Phase 3: MVP 的阶段性初始化 (RSI)
针对“移动抓取”任务，简化初始化阶段，专注攻克抓取难点：

- **阶段 1 `loco` (40%)**：远距离随机出生，手臂收拢。主要训练底盘导航。
- **阶段 2 `approach` (30%)**：出生在距离物体 0.5m 处，手臂开始伸展。主要训练底盘微调与手臂对准。
- **阶段 3 `pre_grasp` (30%)**：**核心难点**。夹爪已经贴近物体两侧，主要训练夹爪的精准闭合与提拉发力。

## 4. 实施路线图 (MVP Roadmap)

| 阶段 | 任务 | 验证标准 | 预期用时 |
| :--- | :--- | :--- | :--- |
| **Milestone 1** | 环境与资产导入 | 轮式机械臂在 Isaac Gym 中加载成功，底盘和手臂能分别响应随机 action。 | 1 天 |
| **Milestone 2** | 短程专家数据采集 | 能够通过 `play_dataset.py` 重放 50 条“移动并抓取”的规划轨迹。 | 1-2 天 |
| **Milestone 3** | 纯 RL 抓取训练 | 暂时关闭 AMP。开启 30% `pre_grasp` 初始化，机器人能够大概率抓起物体。 | 2 天 |
| **Milestone 4** | AMP 风格对齐 | 开启 AMP。机器人的抓取动作变得连贯、无剧烈抖动，且底盘移动时不发生明显甩臂。 | 2 天 |
