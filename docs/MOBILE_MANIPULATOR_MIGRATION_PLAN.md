# PhysHSI 架构迁移计划：从双足人形到移动轮式机械臂

## 1. 迁移目标与核心方法论

**目标**：将现有的 PhysHSI 强化学习框架（基于 `legged_gym` 和 `rsl_rl`）迁移至“移动轮式底盘 + 单个机械臂”的硬件架构上，完成类似 `carrybox` 的人机物交互任务。

**保留的核心方法论**：
1. **AMP (Adversarial Motion Priors)**：使用对抗性先验，约束机器人生成平滑、协同的（非生硬的）控制策略。
2. **阶段性状态初始化 (Stage-wise Initialization / RSI)**：通过在任务的不同阶段进行初始化，解决长程 (Long-horizon) 任务梯度稀疏的问题。

---

## 2. 核心需求与准备工作清单

### 2.1 硬件/仿真资产 (Assets)
- [ ] **URDF 文件**：需要包含轮式底盘（通常使用连续关节 `continuous` 模拟轮子或直接使用虚根节点 `Virtual Root` 控制位移）和单机械臂（如 UR5, Franka 等）。
- [ ] **关节映射定义**：整理新机器人的 DOF 列表，明确区分哪些关节属于底盘控制，哪些属于机械臂控制。

### 2.2 专家数据 (Expert Data) —— AMP 迁移的关键
由于失去了人类全身动捕数据，必须重新构建参考轨迹库。
- **需求**：至少 50-100 条包含 `[底盘位姿, 机械臂关节角, 末端执行器位姿, 目标物体状态]` 的时间序列数据。
- **获取途径建议**：
  1. **脚本/规划器生成**：使用 ROS/MoveIt! 规划机械臂轨迹，结合底盘的 A* 路径规划，录制物理状态。
  2. **遥操作采集 (Teleop)**：使用手柄或 VR 设备人工控制机器人完成任务并记录。
- **数据格式化**：编写脚本将上述轨迹转换为与 `legged_gym/envs/motionlib` 兼容的张量格式。

## 3. 代码重构细节 (Refactoring Details)

### Phase 1: 基础环境剥离
在 `legged_gym/envs/` 下创建新的机器人目录（如 `envs/mobile_manipulator/`）。

1. **`base_task.py` 与 `legged_robot.py`**：
   - 移除所有关于“脚部接触 (feet contact)”、“空中时长 (air time)”、“双腿对称性”的硬编码逻辑。
   - 保留物理引擎初始化、观测值拼接和 PPO 缓冲区的分配。

2. **动作空间重构 (`_compute_torques`)**：
   - **机械臂**：继续使用 PD 控制（高刚度 `P`，适中 `D`）。
   - **轮式底盘**：如果底层是速度控制，需将输出直接映射为底盘的目标线速度/角速度（而不是输出扭矩）。

### Phase 2: 观测空间与奖励函数 (Obs & Rewards)

1. **观测空间 (Observation Space)**
   - **减项**：去除 29 自由度的关节位置/速度。
   - **增项**：加入底盘的相对位移、末端夹爪 (Gripper) 的状态（开/关）、夹爪相对于目标物体的精确距离。

2. **AMP 判别器输入 (AMP Observations)**
   - 需要重新定义“什么是好的动作”。输入应包括：机械臂关节速度的平滑度、底盘移动与机械臂运动的协同度（例如：底盘转弯时，手臂是否能做出反向补偿）。

3. **任务奖励函数 (Task Rewards)**
   - **追踪目标**：底盘移动至目标点。
   - **抓取奖励**：夹爪闭合且成功限制住目标物体的自由度。
   - **奇异性惩罚**：强烈惩罚机械臂到达关节极限或奇异点。
   - **平滑度惩罚**：惩罚底盘的急刹车或手臂的急剧抽动。

### Phase 3: 继承阶段性初始化 (Stage-wise Initialization)
完全复用我们在 G1 上成功的经验，重构 `carrybox_config.py` 中的 `reset_mode`：

- **阶段 1 (`loco`)**：随机位置出生，手臂归位。
- **阶段 2 (`pickUp`)**：出生在物体前方，底盘静止，手臂处于准备抓取状态。
- **阶段 3 (`carryWith`)**：夹爪已闭合抓取物体，随机放置在地图中。
- **阶段 4 (`putDown`)**：**关键点**。抓着物体，直接出生在目标放置点（如桌子/架子）前方。

## 4. 实施路线图 (Roadmap)

| 阶段 | 任务 | 验证标准 | 预期用时 |
| :--- | :--- | :--- | :--- |
| **Milestone 1** | 环境搭建与 URDF 导入 | 机器人在 Isaac Gym 中不报错，能通过随机 action 移动并挥舞手臂。 | 1-2 天 |
| **Milestone 2** | 专家数据采集与格式化 | 能够通过 `play_dataset.py` 在仿真中重放规划器/人工控制的平滑轨迹。 | 2-3 天 |
| **Milestone 3** | 第一阶段纯 RL 训练 | 暂时关闭 AMP。机器人能完成初步的“走向并抓住物体”（动作可能生硬）。 | 3-4 天 |
| **Milestone 4** | 引入 AMP 与阶段初始化 | 开启 AMP 和 30% `putDown` 初始化。机器人能够平滑、连贯地完成拿取和放置。 | 2-3 天 |
