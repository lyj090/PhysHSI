# Carrybox 训练进阶报告：第二阶段 (放置动作强化)

## 1. 第一阶段 (Iteration 20,000) 现状分析

在完成了基础的 20,000 次迭代训练后，机器人表现出以下特征：
- **优点**：能够极稳健地走向箱子、抓取并进行长距离负重行走（Episode Length ~800）。
- **瓶颈**：**“只会搬，不会放”**。到达目标点后，机器人倾向于继续抱着箱子原地踏步，无法触发最终的放置逻辑。
- **根本原因**：由于搬运路径过长，机器人在随机探索中极难偶然触发“成功放置”的正向反馈，导致该子任务的梯度信息不足。

## 2. 第二阶段策略：论文级“阶段性初始化” (Stage-wise Initialization)

为了解决长程任务的逻辑闭环问题，我们从手动调权转向了 PhysHSI 论文中推荐的**阶段性初始化策略**。

### 2.1 核心技术调整
我们将环境的初始状态分布（`skill_init_prob`）进行了重构，不再强制要求机器人每次都从零开始：

| 阶段 (Skill) | 初始概率 (Old) | **初始概率 (New)** | 状态解读 |
| :--- | :--- | :--- | :--- |
| **loco** (行走) | 0.8 | **0.4** | 从空手站立开始。 |
| **pickUp** (拿起) | 0.2 | **0.1** | 出生在箱子前，准备抓取。 |
| **carryWith** (搬运) | 0.2 | **0.2** | 直接抱着箱子出生。 |
| **putDown** (放下) | 0.0 | **0.3** | **核心改进**：直接抱着箱子出生在目标点附近。 |

### 2.2 预期效果
通过将 **30%** 的环境设为 `putDown` 初始化，机器人可以：
1. **跳过漫长的搬运过程**，直接在高频次下练习“如何把箱子放到架子上”。
2. **打通反向链条**：当它学会了“放”，`carryWith` 阶段就能顺着这个成功的路径学会如何“走过去”，从而实现全链路连通。
