# 08 迁移指南：从 PhysHSI 到轮式底盘+单臂

> **文档定位**：将 PhysHSI 的 AMP 训练管线迁移到你的轮式底盘+单机械臂平台，在 Isaac Sim/Lab 中实现物理仿真+AMP 风格约束的操作策略。
>
> **前置**：[[01-Projects/AMP-Reproduction/PhysHSI/01-PhysHSI总览与论文对照|01-07 全部文档]]
> **目标平台**：Isaac Sim/Lab + 轮式差速底盘 + 6-7 DOF 机械臂 + 夹爪

---

## 1. 总体迁移策略

### 1.1 分阶段路线

```
阶段 A (2 周): Isaac Sim/Lab 基础环境
  └─ 导入机器人 URDF + 场景搭建 + PD 控制验证

阶段 B (2 周): 纯任务 RL（无 AMP）
  └─ PPO 训练 pick-and-place，验证 reward 设计

阶段 C (3 周): 加入 AMP
  └─ 录制专家轨迹 → 训练 AMP 判别器 → HIM PPO 联合训练

阶段 D (后续): 域随机化 + sim-to-real 准备
  └─ 全面域随机化 → 实机测试
```

### 1.2 技术栈对比

| 组件 | PhysHSI | 你的平台 |
|------|---------|---------|
| **仿真引擎** | Isaac Gym | Isaac Sim / Isaac Lab |
| **物理后端** | PhysX (Isaac Gym API) | PhysX (USD Physics) |
| **环境框架** | legged_gym (自建) | Isaac Lab (官方) |
| **RL 算法** | HIM PPO (rsl_rl 魔改) | 可直接复用 HIM PPO |
| **机器人** | G1 人形 (29 DOF) | 轮式底盘+单臂 (~10 DOF) |
| **URDF/资产** | `.urdf` | `.usd` (推荐) 或 `.urdf` |
| **并行环境** | GPU tensorized | Isaac Lab `ManagerBasedRLEnv` |

## 4. 观测空间重新设计

### 4.1 本体感知观测

```python
# 对应 PhysHSI 的 current_actor_obs
proprio_obs = torch.cat([
    base_ang_vel * 0.25,              # [3] 底盘角速度
    projected_gravity,                 # [3] 重力方向（IMU）
    (arm_joint_pos - default_pos),    # [6] 机械臂关节偏差
    arm_joint_vel * 0.05,            # [6] 机械臂关节速度
    end_effector_pos - base_pos,      # [3] 末端相对位置
    gripper_state,                    # [1] 夹爪开度
    last_actions,                     # [9] 上一时刻动作
    base_lin_vel * 2.0,             # [3] 底盘线速度（privileged可给）
], dim=-1)  # 约 34 维（vs G1的108维）
```

### 4.2 任务观测

```python
# 对应 PhysHSI 的 compute_task_observations
task_obs = torch.cat([
    # 物体信息
    object_pos - base_pos,            # [3] 物体相对位置
    object_rot_6d,                    # [6] 物体姿态 6D
    # 目标信息
    goal_pos - base_pos,              # [3] 目标相对位置
    # 物体尺寸（随机化时变化）
    object_size,                      # [3] 物体尺寸
], dim=-1)  # 15 维
```

### 4.3 感知噪声模型

```python
# 直接复用 PhysHSI 的三模式设计
if add_noise:
    # 粗粒度: 距离 > 1.0m（看不到标签）
    is_coarse = distance > thresh_tag

    # 遮挡: 距离 < 0.65m 但视野受阻
    is_mask = (~can_see_tag) & has_seen_tag & (distance < 0.65)

    # 注入噪声
    object_pos += rand_pos_noise(σ=0.05)
    object_rot += rand_rot_noise(σ=5°)

    # 粗粒度: 用粗略估计
    object_pos[is_coarse] = coarse_estimate + offset
    # 遮挡: 用最后已知位置
    object_pos[is_mask] = last_seen_pos
```

## 5. AMP 适配

### 5.1 判别器网络

```python
# 网络结构完全复用！只需调整输入维度
# PhysHSI: num_obs = 600 (10帧 × 60维)
# 你的: num_obs = 10 × 帧维度

# 单帧 AMP 观测
amp_frame_dim = (
    3 +    # base_ang_vel
    6 +    # arm_joint_pos
    6 +    # arm_joint_vel
    3 +    # end_effector_pos (相对基座)
    3 +    # object_pos (局部系)
    3 +    # base_lin_vel
    6      # base_rot_6d
)  # = 30 维

amp_window = 10  # 10帧
num_amp_obs = 30 * 10 = 300

amp = AMP(
    num_obs=300,                    # ← 唯一需要改的维度
    amp_coef=0.25,
    hidden_dims=[512, 256],         # 保持不变
    activation='relu',
    device='cuda:0'
)
```

### 5.2 Style Reward

```python
# 完全复用 PhysHSI 的 reward 函数
def compute_style_reward(amp_obs):
    with torch.no_grad():
        amp.eval()
        d = amp(amp_obs)
        amp.train()
        # 相同公式：r = max(0, 1 - 0.25*(D-1)²)
        return torch.clamp(1 - 0.25 * (d - 1).pow(2), min=0)

# 相同权重
total_reward = 0.25 * style_reward + 0.75 * task_reward
```

### 5.3 专家数据来源

**PhysHSI**：mocap 重定向 → G1 关节序列

**你的平台**：需要录制专家轨迹

## 6. 奖励函数适配

### 6.1 任务分解

```
阶段 1: 导航靠近 (nav_to_object)
  ├─ base_to_object_vel:    底盘朝向物体速度
  ├─ heading_to_object:     朝向对齐
  └─ base_stability:        底盘稳定（惩罚角速度）

阶段 2: 抓取 (grasp)
  ├─ ee_to_object:          末端接近物体
  ├─ gripper_closure:       夹爪闭合度
  ├─ object_lifted:         物体离地高度
  └─ contact_force:         接触力（不宜过大）

阶段 3: 搬运放置 (carry_and_place)
  ├─ base_to_goal_vel:      底盘朝向目标速度
  ├─ ee_holding_object:     末端保持抓取
  ├─ object_to_goal:        物体接近目标
  └─ place_on_goal:         物体在目标上方

阶段 4: 释放 (release)
  ├─ gripper_open:          夹爪打开
  ├─ ee_retract:            末端撤回
  └─ object_stable:         物体稳定在目标
```

### 6.2 奖励尺度参考

```python
reward_scales = {

    'object_to_goal': 1.5,      # 搬运主奖励

    'gripper_open': 0.5,

}
```

## 7. HIM PPO 直接复用

因为 HIM PPO 算法与环境无关（只依赖 obs/action 维度），可以**几乎直接复用**：

```python
# 唯一需要修改的地方：
actor_critic = ActorCritic(
    num_actor_obs=actor_obs_length,   # 你的观测维度
    num_critic_obs=privileged_obs_length,
    num_actions=9,                    # 你的动作维度
    actor_hidden_dims=[512, 256, 128],
    critic_hidden_dims=[512, 256, 128],
    activation='elu',
)

him_ppo = HIMPPO(
    actor_critic=actor_critic,
    amp=amp,
    amp_normalizer=None,  # 初始关闭
    motion_buffer=motion_buffer,
    value_smoothness_coef=0.1,
    use_muon_optim=False,
    # 其余全部保持默认
)
```

## 10. 关键差异总结

| 维度 | PhysHSI (G1) | 你的平台 | 复用程度 |
|------|-------------|---------|---------|
| AMP 判别器 | MLP(600→512→256→1) | MLP(300→512→256→1) | 架构复用，改输入维度 |
| HIM PPO | 完整实现 | 直接复用 | **100% 复用** |
| Style Reward | max(0,1-0.25(D-1)²) | 相同 | **100% 复用** |
| Smoothness Reg | ✓ | ✓ | **100% 复用** |
| PD 控制 | 29 DOF 分组 | ~10 DOF 分组 | 架构复用，参数调整 |
| MotionLib | 全身 mocap | 末端/关节轨迹 | 架构复用，数据重录 |
| 观测噪声 | 三模式 | 三模式 | **100% 复用** |
| 域随机化 | 全面 | 全面 | **100% 复用** |
| 多阶段训练 | 4 阶段（loco→pick→carry→put） | 3 阶段（nav→grasp→place） | 架构复用，阶段简化 |
| 仿真引擎 | Isaac Gym API | Isaac Lab API | **不兼容**，需重写环境 |

**估计工作量**：
- 环境重写（Isaac Gym → Isaac Lab）：**~60%** 的工作量
- AMP/HIM PPO 算法部分：**~5%** 的工作量（几乎直接复用）
- 专家数据录制与适配：**~20%** 的工作量
- 奖励函数重设计：**~15%** 的工作量
