## 1. 训练入口

### 1.1 训练脚本

```python
# legged_gym/legged_gym/scripts/train.py (48 行，极简设计)

from legged_gym.envs import *
from legged_gym.utils import get_args, task_registry

def train(args, headless=False):
    # 1. 创建环境（通过任务注册表）
    env, env_cfg = task_registry.make_env(name=args.task, args=args)

    # 2. 创建算法 Runner（HIM PPO）
    ppo_runner, train_cfg = task_registry.make_alg_runner(
        env=env, name=args.task, args=args)

    # 3. 开始训练
    ppo_runner.learn(
        num_learning_iterations=train_cfg.runner.max_iterations,
        init_at_random_ep_len=True
    )

if __name__ == '__main__':
    args = get_args()
    train(args)
```

**命令行参数**（`get_args()`）：
```bash
--task carrybox          # 任务名
--headless               # 无渲染训练
--resume                 # 从 checkpoint 恢复
--resume_path PATH       # checkpoint 路径
--experiment_name NAME   # 实验名（用于日志目录）
--run_name NAME          # 运行名
```

### 1.2 任务注册表

```python
# legged_gym/utils/task_registry.py

def make_env(name, args):
    # 1. 根据 name 查找配置类
    task_to_cfg = {
        'carrybox': (G1Cfg, G1CfgPPO),
        'carrybox_resume': (G1CfgResume, G1CfgPPOResume),
        'sitdown': (G1SitDownCfg, G1SitDownCfgPPO),
        'liedown': (G1LieDownCfg, G1LieDownCfgPPO),
        'standup': (G1StandUpCfg, G1StandUpCfgPPO),
        'styleloco_dinosaur': (G1StyleLocoDinosaurCfg, ...),
        'styleloco_highknee': (G1StyleLocoHighkneeCfg, ...),
    }

    # 2. 实例化环境
    env_cfg, train_cfg = task_to_cfg[name]
    env = LeggedRobot(cfg=env_cfg(), sim_params=..., ...)

    return env, env_cfg

def make_alg_runner(env, name, args):
    # 1. 创建 ActorCritic 网络
    actor_critic = ActorCritic(
        num_actor_obs=env.num_actor_obs,
        num_critic_obs=env.num_privileged_obs,
        num_actions=env.num_actions,
        ...
    )

    # 2. 创建 AMP 判别器
    amp = AMP(num_obs=env.num_amp_obs, amp_coef=cfg.amp.amp_coef, ...)

    # 3. 创建 HIM PPO
    algorithm = HIMPPO(actor_critic=actor_critic, amp=amp, ...)

    # 4. 创建 Runner（含训练循环、日志、保存）
    runner = OnPolicyRunner(env, algorithm, train_cfg, ...)

    return runner, train_cfg
```

## 2. 训练循环内部

### 2.1 Runner 主循环

```python
# 伪代码：OnPolicyRunner.learn()

def learn(self, num_learning_iterations, init_at_random_ep_len=True):
    obs = self.env.get_observations()
    critic_obs = obs  # 或 privileged_obs

    for it in range(num_learning_iterations):
        # === Rollout Phase ===
        for step in range(self.num_steps_per_env):  # 100 steps
            # 1. 采样动作
            actions = self.alg.act(obs, critic_obs)

            # 2. 环境步进
            obs, privileged_obs, rewards, dones, infos, \
                termination_ids, termination_obs, amp_obs = self.env.step(actions)

            # 3. 融合 AMP style reward
            amp_reward = self.amp.predict_reward(amp_obs, self.amp_normalizer)
            rewards = self.amp.combine_reward(amp_reward, rewards, stage)

            # 4. 存储 transition
            self.alg.process_env_step(rewards, dones, infos, privileged_obs)
            self.alg.process_amp_state(amp_obs)

            # 5. 处理终止环境的观测（用于 GAE bootstrapping）
            if len(termination_ids) > 0:
                self._handle_terminations(termination_ids, termination_obs)

        # === Update Phase ===
        # 6. 计算回报
        self.alg.compute_returns(last_critic_obs)

        # 7. PPO 更新（含 AMP 联合训练）
        value_loss, surrogate_loss, amp_loss, expert_loss, policy_loss \
            = self.alg.update()

        # 8. 日志记录
        if it % self.log_interval == 0:
            self.log({
                'value_loss': value_loss,
                'surrogate_loss': surrogate_loss,
                'amp_loss': amp_loss,
                'expert_loss': expert_loss,
                'policy_loss': policy_loss,
                'mean_reward': mean_reward,
                'mean_episode_length': mean_ep_len,
                ...
            })

        # 9. Checkpoint 保存
        if it % self.save_interval == 0:
            self.save(os.path.join(self.log_dir, f'model_{it}.pt'))
```

### 2.2 单次 Update 的损失曲线示意

```
训练中应观察的趋势：
  value_loss        ↓ 下降并稳定
  surrogate_loss    ↓ 下降并稳定  
  amp_loss          ↓ 判别器收敛（区分力下降）
  expert_loss       → 接近 0（专家正确分类）
  policy_loss       → 接近 0（策略被正确识别）
  mean_reward       ↑ 上升
  mean_ep_length    → 接近 max_episode_length
```

## 3. Checkpoint 管理

### 3.1 保存内容

```python
def save(self, path):
    torch.save({
        'model_state_dict': self.actor_critic.state_dict(),
        'amp_state_dict': self.amp.state_dict(),
        'optimizer_state_dict': self.optimizer.state_dict(),
        'amp_normalizer': self.amp_normalizer,
        'iter': self.current_iter,
        'env_cfg': self.env_cfg,
        'train_cfg': self.train_cfg,
        'amp_obs_normalizer_state': ...,  # 可选
    }, path)
```

### 3.2 恢复训练

```python
def load(self, path):
    ckpt = torch.load(path)
    self.actor_critic.load_state_dict(ckpt['model_state_dict'])
    self.amp.load_state_dict(ckpt['amp_state_dict'])
    self.optimizer.load_state_dict(ckpt['optimizer_state_dict'])
    self.current_iter = ckpt['iter']
    # 注意：env_cfg 从 checkpoint 恢复，保证一致性
```

## 6. ONNX 导出（部署用）

```python
# legged_gym/utils/helpers.py

def export_policy_as_onnx(actor_critic, path, device='cpu'):
    """导出策略为 ONNX 格式，用于实机部署"""
    actor_critic.to(device)
    actor_critic.eval()

    dummy_input = torch.randn(1, actor_critic.num_actor_obs, device=device)

    torch.onnx.export(
        actor_critic.actor,          # 仅导出 actor（策略网络）
        dummy_input,
        path,
        export_params=True,
        opset_version=11,
        input_names=['observations'],
        output_names=['actions'],
        dynamic_axes={'observations': {0: 'batch_size'},
                      'actions': {0: 'batch_size'}}
    )
```

## 7. 常见问题与调试

### 7.1 NaN 动作

```python
# him_ppo.py: act() 中的防护
if obs.isnan().any():
    obs = torch.zeros((obs.shape[0], obs.shape[1]), device=obs.device)
    critic_obs = torch.zeros(...)
```

**原因**：环境刚重置时 buffer 未填充，历史观测含 NaN。

### 7.2 训练不稳定

- **降低学习率**：`1e-3 → 5e-4`
- **增大 clip_param**：`0.2 → 0.3`
- **关闭 adaptive KL**：`schedule='fixed'`
- **检查 reward 尺度**：正则化奖励不应主导任务奖励

### 7.3 AMP 判别器过强

**现象**：`expert_loss ≈ 0, policy_loss ≈ 0`，但 style reward 恒为 0。

**原因**：判别器太快收敛，策略学不到任何 style 信号。

**修复**：
- 减小 `amp_coef`（0.25 → 0.1）
- 增加判别器 weight decay（`10e-4 → 10e-3`）
- 在 AMP 观测中注入噪声

### 7.4 前几个 episode 穿透

> 已知问题：play.py 的头 1-2 个 episode 可能看到机器人与物体轻微穿插。不影响训练，之后的 episode 正常。

从仿真训练到实机部署的检查项：

- [ ] ONNX/JIT 导出策略权重
- [ ] 验证推理延迟（目标 < 2ms on embedded GPU）
- [ ] 确认观测预处理一致（归一化参数对齐）
- [ ] 域随机化范围对标真实环境噪声
