## 1. 环境架构总览

### 1.1 继承层次

```
BaseTask (base_task.py)
  └── LeggedRobot (carrybox.py)
        ├── CarryBox
        ├── SitDown
        ├── LieDown
        ├── StandUp
        └── StyleLoco
```

- `BaseTask`：管理 Isaac Gym 仿真生命周期（创建/销毁/渲染）
- `LeggedRobot`：添加机器人特有的逻辑（PD 控制、观测构建、终止检测、域随机化）
- `具体任务类`：定义任务专属的奖励函数、观测空间、重置逻辑

### 1.2 并行仿真架构

```
┌──────────────────────────────────────────┐
│              Isaac Gym GPU               │
│  ┌────┐ ┌────┐ ┌────┐      ┌────┐     │
│  │Env0│ │Env1│ │Env2│ ...  │EnvN│     │  N ≤ 4096
│  └────┘ └────┘ └────┘      └────┘     │
│     │      │      │            │        │
│     └──────┴──────┴────────────┘        │
│                    │                     │
│         ┌──────────▼──────────┐         │
│         │  Tensorized State   │         │
│         │  [N, D] on GPU      │         │
│         └──────────▼──────────┘         │
│                    │                     │
│         ┌──────────▼──────────┐         │
│         │  Policy Network     │         │
│         │  → Action [N, A]    │         │
│         └──────────▼──────────┘         │
│                    │                     │
│         ┌──────────▼──────────┐         │
│         │  Physics Step       │         │
│         │  (GPU accelerated)  │         │
│         └─────────────────────┘         │
└──────────────────────────────────────────┘
```

**关键机制**：所有环境的状态存储为 GPU Tensor，策略推理和物理步进都是批量操作——单次前向传播处理 N 个环境。

### 1.3 仿真步进流程

```python
# carrybox.py: step()
def step(self, actions):
    clip_actions = self.cfg.normalization.clip_actions
    self.actions = torch.clip(actions, -clip_actions, clip_actions)

    self.render()
    for _ in range(self.cfg.control.decimation):  # decimation=4
        # 1. 计算关节力矩（PD 控制）
        self.torques = self._compute_torques(self.actions)
        # 2. 设置驱动力
        self.gym.set_dof_actuation_force_tensor(self.sim, unwrap(self.torques))
        # 3. 物理步进（sim DT = 0.005s）
        self.gym.simulate(self.sim)
        if self.device == 'cpu':
            self.gym.fetch_results(self.sim, True)
        # 4. 刷新 DOF 状态
        self.gym.refresh_dof_state_tensor(self.sim)

    termination_ids, termination_obs, amp_obs = self.post_physics_step()
    return obs, privileged_obs, reward, dones, extras, termination_ids, termination_obs, amp_obs
```

**步进时序**（以 CarryBox 为例）：

```
Policy DT = 0.02s (50Hz)  ← decimation=4, sim DT=0.005s
┌──────┬──────┬──────┬──────┐
│ PD 0 │ PD 1 │ PD 2 │ PD 3 │  ← 4次PD控制步
│5ms   │5ms   │5ms   │5ms   │
└──────┴──────┴──────┴──────┘
          ↓
    一次 policy 推理
```

## 2. MotionLib 运动库系统

### 2.1 架构设计

MotionLib 是 AMP 的"专家数据源"——存储参考运动帧，提供采样接口。

```
Reference motion files (.pt)
  │
  ├─ carrybox/     ← 搬箱子动作序列
  ├─ sitdown/      ← 坐下动作序列
  ├─ liedown/      ← 躺下动作序列
  ├─ standup/      ← 站起动作序列
  └─ styleloco/    ← 风格化行走序列
  │
  ▼
MotionLib.__init__()
  │
  ├─ load_motions()     ← 读取 YAML 配置 + .pt 文件
  ├─ process_motions()  ← 计算基座/关节/末端轨迹
  └─ init_amp_buffers() ← 构建 AMP 观测窗
  │
  ▼
get_expert_obs(batch_size)  ← 训练时采样
  │
  ├─ 采样运动文件（按权重）
  ├─ 随机选择起始帧（含 RSI 范围约束）
  └─ 返回多帧堆叠的 AMP 观测 [B, window × frame_dim]
```

### 2.2 运动文件结构

```yaml
# resources/config/carrybox.yaml
motions:
  loco:                          # 技能名
    - file: dataset/carrybox/loco_001.pt
      weight: 1.0                # 采样权重
      rsi_skipped_range: []      # RSI 跳过的帧范围

    - file: dataset/carrybox/loco_002.pt
      weight: 1.5
      rsi_skipped_range: [0, 100]  # 前100帧不用于RSI

  pickUp:
    - file: dataset/carrybox/pickup_001.pt
      weight: 2.0                # 更高权重 → 更多采样
      rsi_skipped_range: [50, 200]

  carryWith:
    - file: dataset/carrybox/carry_001.pt
      weight: 1.0

  putDown:
    - file: dataset/carrybox/putdown_001.pt
      weight: 1.0
      rsi_skipped_range: [300, 500]
```

### 2.3 运动数据处理

```python
def process_motions(self, dof_names):

    self.motion_base_rpy = ...      # 基座 roll/pitch/yaw
    self.motion_base_lin_vel = ...  # 基座局部线速度
    self.motion_base_ang_vel = ...  # 基座局部角速度
    self.motion_global_lin_vel = ...# 基座全局线速度
    self.motion_global_ang_vel = ...# 基座全局角速度
    self.motion_dof_pos = ...       # 关节位置 [tot_frames, num_dof]
    self.motion_dof_vel = ...       # 关节速度
    self.motion_end_effector_pos = ...  # 末端位置

    # 构建 AMP 观测窗口（滑动窗口堆叠）
    for frame in range(tot_frames):
        amp_obs_window = []

            idx = max(0, frame - w)  # 向前追溯
```

### 2.4 专家观测采样

```python
def get_expert_obs(self, batch_size):
    # 1. 按权重采样运动文件
    motion_ids = torch.multinomial(self.motion_weights, batch_size, replacement=True)

    # 2. 对每个采样，随机选择起始帧
    frames = torch.randint(0, motion_lengths, (batch_size,))

    # 3. 提取预计算的 AMP 观测
    expert_obs = self.amp_obs[global_frame_indices]

    return expert_obs  # [batch_size, window_length * frame_dim]
```

### 2.5 RSI（Randomized State Initialization）

RSI 的核心思想：不从固定初始状态开始训练，而是从参考运动的随机帧初始化机器人状态。

```python
# 配置
ratio_random_range: [0.95, 1.05]  # 时间比例随机抖动
rsi_skipped_range: [start, end]   # 该技能中不适合 RSI 的帧范围

# 使用
def reset_with_rsi(self, env_ids):
    # 1. 随机选运动文件
    # 2. 随机选帧（跳过 rsi_skipped_range）
    # 3. 将机器人设置为该帧的状态
    self.dof_pos[env_ids] = motion_dof_pos[frame]
    self.dof_vel[env_ids] = motion_dof_vel[frame]
    self.root_states[env_ids, :3] = motion_base_pos[frame]
    self.root_states[env_ids, 3:7] = motion_base_quat[frame]
    # 4. 调整时间比例（±5% 随机）
    ratio = uniform(0.95, 1.05)
    self.dof_vel[env_ids] *= ratio
    self.base_lin_vel *= ratio
```

> **RSI 的作用**：课程学习——早期从容易的帧开始（如站姿），逐渐覆盖完整运动序列，提升对罕见状态的覆盖。

## 4. 仿真参数配置

```python
# carrybox_config.py
class sim:
    dt = 0.005           # 物理步长 5ms (200Hz)
    substeps = 1         # 每步的子步数
    gravity = [0., 0., -9.81]

    class physx:
        num_threads = 10
        solver_type = 1         # TGS (Temporal Gauss-Seidel)
        num_position_iterations = 8
        num_velocity_iterations = 0
        contact_offset = 0.01   # 接触检测距离
        rest_offset = 0.0
        max_gpu_contact_pairs = 2^24
```

**Policy DT 计算**：
```
policy_dt = sim.dt × decimation = 0.005 × 4 = 0.02s (50Hz)
episode_length = 20s / 0.02s = 1000 steps
```

## 5. 相机与视觉感知

PhysHSI 在机器人头部安装了一个虚拟相机传感器（模拟 Intel D455 深度相机）：

```python
def create_cameras(self):
    self.camera_params = gymapi.CameraProperties()
    self.camera_params.width = 640
    self.camera_params.height = 480
    self.camera_params.horizontal_fov = 87.5  # 随机化范围 [85°, 90°]
    self.camera_params.enable_tensors = True

    for env_handle in self.envs:
        camera_handle = self.gym.create_camera_sensor(env_handle, camera_params)

```

**相机在 CarryBox 中的作用**：
- 检测视野中是否能看到目标标签（AprilTag）
- 提供物体位置的粗略估计（远距离时）
- 观测中加入噪声模拟真实感知不确定性

## 7. 配置系统

PhysHSI 使用深度嵌套的配置类，通过递归实例化自动展开：

```python
# base_config.py
class BaseConfig:
    def __init__(self, update=None):
        for key, value in self.__class__.__dict__.items():
            if not key.startswith('__'):
                if isinstance(value, type):
                    # 递归创建子配置
                    setattr(self, key, value())
        if update:
            self.__dict__.update(update)

# legged_robot_config.py
class LeggedRobotCfg(BaseConfig):
    class env(BaseConfig): ...
    class init_state(BaseConfig): ...
    class control(BaseConfig): ...
    class asset(BaseConfig): ...
    class domain_rand(BaseConfig): ...
    class rewards(BaseConfig): ...
    class normalization(BaseConfig): ...
    class noise(BaseConfig): ...
    class viewer(BaseConfig): ...
    class sim(BaseConfig): ...

# carrybox_config.py
class G1Cfg(LeggedRobotCfg):
    class env(LeggedRobotCfg.env):
        num_envs = 1024
        num_actions = 29
        ...
```

**使用方式**：
```python
cfg = G1Cfg()
# cfg.env.num_envs → 1024
# cfg.control.stiffness → {'hip_yaw': 150, ...}
```
